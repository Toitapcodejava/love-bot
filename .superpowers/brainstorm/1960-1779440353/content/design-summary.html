Design Tổng Thể — Connected State System

Tóm tắt toàn bộ những gì sẽ build. Approve để đi vào implementation plan.

1. Đổi tên nhân vật

AI (trước)
🌙 "Kem" (header chat)
→
AI (mới)
🌙 "Bạn của Kem"

User = "Ngọc Khánh" / "Kem". Chat bubbles user side không cần label, nhưng greeting message AI sẽ gọi đúng tên "Kem" hoặc "Ngọc Khánh".

2. Check-in Ritual

  • Hiện khi mở app lần đầu trong ngày (hoặc sau 4h không dùng)
  • Grid 6 emoji trạng thái + optional text note
  • Lưu vào userStatus (AsyncStorage): { emoji, label, note, timestamp }
  • AI nhận status qua system prompt context

3. Status Chip (Chat Screen)

  • Floating chip ngay trên input bar, hiển thị emoji + label hiện tại
  • Tap chip → mở mini picker để đổi status bất kỳ lúc nào
  • Nếu status = tức/buồn/lo/trống/kiệt → hiện thêm chip "🔥 Đập ngay" → navigate sang Rage screen với boss đúng loại

4. Boss Battle (Rage Screen)

  • Boss type auto-select từ userStatus, hoặc Kem tự chọn nếu không có status
  • HP bar, idle float animation, weapon selector (giữ nguyên 4 weapons cũ)
  • Touch anywhere trên boss area → damage hit, shake animation, damage number float
  • Combo multiplier: đánh liên tục <1s tăng x, nghỉ >1.5s reset
  • HP < 30%: boss đổi màu đỏ, rung nhanh hơn
  • Boss chết: explosion → death screen → AI quip → nút "Nói chuyện" về chat

Luồng kết nối

Check-in → userStatus store → Status chip + AI context
↓ (nếu mood tiêu cực)
"Đập ngay" chip → Boss Battle → Quip → Chat

Ngoài scope (không build lần này)

XP system, achievements, level-up, push notifications, history check-in
